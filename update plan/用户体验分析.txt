================================================================================
            龙门币凑数计算器 2.0 改造计划 —— 角度二：用户体验分析
================================================================================


【一、导航与布局】

1. Sidebar 在每个页面重复定义了 4 份（严重）
   现状：App.js、Note.js、Data.js、About.js 各自定义了一份 Sidebar 组件，
   且实现方式不完全一致（Note.js 和 About.js 是手写的 div 列表，
   Data.js 和 App.js 用了 map 遍历）。
   如果要加一个新页面或改导航项，需要改 4 个文件。

   建议：抽取为全局共享的 Layout 组件，所有页面共用同一个 Sidebar。

2. 没有当前页面的高亮指示
   现状：用户在任何页面上，Sidebar 的四个链接看起来完全一样，
   无法直观知道自己当前在哪个页面。

   建议：使用 React Router 的 NavLink 组件或 useLocation，
   为当前页面的链接添加 active 样式。

3. 移动端导航体验差
   现状：800px 以下 Sidebar 变成水平排列的 4 个按钮，
   占据屏幕顶部空间，且没有折叠/展开功能。
   在小屏幕上，"凑数计算器"标题直接隐藏了（display: none）。

   建议：改为汉堡菜单（hamburger menu），点击展开侧边抽屉或下拉菜单。

4. Footer 与 Sidebar 宽度不匹配（bug）
   现状：Sidebar 宽度是 230px，但 footer 的 margin-left 是 200px，
   导致 footer 和主内容区域左边界不对齐。

   建议：统一为同一个变量/值。


【二、输入体验】

1. 缺少"交换"按钮
   现状：用户输入当前龙门币和目标龙门币后，如果发现填反了，
   需要手动清空两个输入框重新输入。

   建议：在两个输入框之间添加一个"⇅ 交换"按钮，一键互换两个值。

2. 缺少"清空/重置"按钮
   现状：没有一键清空所有输入和设置的功能。
   用户想重新开始时，需要逐个清空输入框。

   建议：添加"重置"按钮，恢复所有输入和设置到默认状态。

3. 限制条件输入框缺少说明
   现状：左侧面板的"允许升级的精零/精一/精二干员数量"和"允许升级的理智的数量"
   四个输入框，placeholder 只写了"0~10"或"0~200"。
   用户不清楚：空着不填代表什么？填 0 代表什么？

   建议：添加提示文字，例如"留空表示不限制"，
   或者用 placeholder 直接写"不限制"。

4. 差值范围限制缺少直观反馈
   现状：差值必须在 -5000~5000 之间，但用户在输入时没有实时提示。
   只有点击"立即计算"后才会看到错误信息。

   建议：在用户输入时实时计算并显示差值，
   如果超出范围则立即用颜色/图标提示，而不是等到点击计算。

5. 输入框宽度固定 280px
   现状：input-box 和 result-box 都是固定 280px 宽度，
   在大屏幕上显得太小，在小屏幕上可能溢出。

   建议：改为响应式宽度（如 width: 100%; max-width: 300px）。


【三、设置面板】

1. 开关文字过长被截断
   现状：toggle-text 设置了 white-space: nowrap + text-overflow: ellipsis，
   在窄屏或右侧面板宽度不够时，长文字会被截断。
   例如"不存在/不使用活动商店1代币换10龙门币"可能只显示一半。
   用户看不到完整描述，就无法理解这个开关的含义。

   建议：允许文字换行（去掉 nowrap），或者用更简短的标签 + tooltip 说明。

2. 开关语义混乱，用户理解成本高
   现状：有的开关是"不允许..."（关闭某功能），有的是"允许..."（开启某功能）。
   用户需要做双重否定的心理运算：
   "不允许使用理智三星通关"开关打开 = 禁止三星通关？
   "允许连续多次对精零1级干员进行升级"开关关闭 = 不允许？

   建议：统一为正向语义，例如"使用理智三星通关：开/关"，
   开关打开 = 允许，关闭 = 不允许，直觉一致。

3. 缺少"恢复默认设置"按钮
   现状：14 个开关没有一键恢复默认的功能。
   用户调整了多个开关后想回到初始状态，需要逐个手动切换。

   建议：在设置面板底部添加"恢复默认"按钮。

4. 缺少预设配置
   现状：不同玩家的游戏阶段不同，常用的设置组合也不同。
   例如新手可能不需要贸易站和基建，老玩家可能只用升级和理智。
   每次都要手动调整多个开关。

   建议：提供几个预设方案，如"仅升级消耗""仅理智消耗""全部开启"等，
   用户一键切换，也可以在预设基础上微调。

5. 设置修改后不会自动重新计算
   现状：用户修改了设置后，需要手动重新点击"立即计算"才能看到新结果。
   这一点虽然在"注意事项"中有说明，但很容易被忽略。

   建议：两种方案——
   a) 设置变更后自动重新计算（如果计算速度够快）
   b) 设置变更后显示醒目提示"设置已更改，请重新计算"


【四、结果展示与路径可视化】

1. 路径结果纯文字，缺乏可视化（核心改进点）
   现状：计算结果以纯文字形式展示，例如：
   "步骤 1：通过【1】次【贸易站售卖2条赤金】，【获得】【1000】个龙门币，
    当前龙门币数量为【1000】个。"
   信息密度高但可读性差，用户需要逐字阅读才能理解。

   建议：
   a) 用卡片式布局展示每个步骤，配合图标/颜色区分获得/花费
   b) 添加进度条或流程图，直观展示龙门币从初始值到目标值的变化过程
   c) 用绿色表示获得、红色表示花费，数字加粗突出
   d) 考虑添加简单的柱状图或折线图展示龙门币变化趋势

2. 缺少路径摘要信息
   现状：用户看到一条路径后，无法快速了解这条路径的"代价"——
   总共需要多少理智？需要多少赤金？需要几个干员？

   建议：在每条路径顶部添加摘要卡片，例如：
   "本方案：共 3 步 | 消耗理智 36 | 需要精零1级干员 2 名 | 合成材料 1 次"

3. 路径切换体验不佳
   现状：只有"上一路径""下一路径"两个按钮和圆点指示器。
   10 条路径需要反复点击才能浏览完。
   圆点指示器在路径多时也不够直观。

   建议：
   a) 改为列表/缩略图模式，一次展示所有路径的摘要，点击展开详情
   b) 或者保留翻页但添加路径编号显示（如"第 3/10 条"）

4. 没有"复制结果"或"分享"功能
   现状：用户想把计算结果分享给朋友，只能截图。

   建议：添加"复制为文字"按钮，将路径方案格式化为可粘贴的文本。
   进阶：生成带参数的分享链接，朋友打开即可看到相同的计算结果。


【五、加载与反馈】

1. 进度条是假的
   现状：计算中显示的进度条只是一个 CSS 动画（translateX 来回移动），
   不反映实际计算进度。用户无法判断还需要等多久。

   建议：至少显示"已等待 X 秒"的计时器，
   或者改为脉冲/骨架屏动画，不暗示"有进度"。

2. 缓存命中时没有反馈
   现状：后端返回了 cache: "hit"/"miss" 字段，
   但前端完全没有利用这个信息。用户不知道结果是缓存的还是新计算的。

   建议：可以在结果区域小字显示"计算耗时 XXms"或"从缓存读取"，
   让用户感知到系统的响应速度。

3. 计算成功/失败缺少明确的视觉反馈
   现状：计算成功后直接显示路径，没有过渡动画或成功提示。
   计算失败时错误信息显示在差值输入框下方，容易被忽略。

   建议：成功时路径区域有淡入动画；
   失败时用 toast 通知或更醒目的错误卡片。


【六、无障碍访问（Accessibility）】

1. 开关控件缺少 ARIA 标签
   现状：14 个 toggle switch 使用的是自定义样式的 checkbox，
   但没有 aria-label 或关联的 <label> 文本。
   屏幕阅读器用户无法理解每个开关的含义。

   建议：为每个 checkbox 添加 aria-label，
   或确保 <label> 的 htmlFor 与 input 的 id 正确关联。

2. 稀有度仅用颜色区分（色盲不友好）
   现状：PathRenderer 中用 getRarityColor 函数为不同稀有度的物品名称
   设置不同颜色（darkgreen、darkblue、purple、orange）。
   色盲用户可能无法区分这些颜色。

   建议：在颜色之外增加文字标记（如星级图标 ★）或不同的字体样式。

3. 弹窗缺少焦点管理
   现状：弹窗（modal）打开时，焦点没有被锁定在弹窗内部，
   用户可以 Tab 到弹窗后面的元素。关闭弹窗后焦点也没有回到触发元素。

   建议：弹窗打开时 trap focus，关闭时恢复焦点。
   支持 Escape 键关闭弹窗。

4. "返回顶部"按钮始终可见
   现状：back-to-top 按钮使用 position: fixed 始终显示在右下角，
   即使用户已经在页面顶部。这浪费了屏幕空间，尤其在移动端。

   建议：只在页面滚动超过一定距离后才显示该按钮。


【七、子页面体验】

1. 注意事项页面（Note）信息过载
   现状：Note 页面是一整页密集的文字说明，包含公式、多组图片、
   大段解释。用户很难快速找到自己需要的信息。

   建议：
   a) 改为折叠式 FAQ 格式，每个问题可展开/收起
   b) 添加目录锚点，用户可以快速跳转到感兴趣的部分
   c) 精简文字，用图表替代大段文字说明

2. 数据页面（Data）的静态数据与 DataService 脱节
   现状：Data.js 中所有表格数据都是硬编码在组件内部的，
   与 DataService.js 中的 classifyData 完全独立。
   如果游戏更新了数值，需要同时修改 DataService 和 Data.js 两处。

   建议：Data 页面的表格数据应该从 DataService 动态生成，
   而不是手动维护一份副本。

3. 数据页面默认全部折叠
   现状：Data 页面的 4 个手风琴面板默认全部折叠，
   用户进入页面后看到的是 4 个标题，需要逐个点击展开。
   对于第一次访问的用户，不知道里面有什么内容。

   建议：默认展开第一个面板，或者在标题下方添加简短的内容预览。

4. 数据页面的"推荐路径快速查找"表格可读性差
   现状：推荐路径表格中，每个单元格包含完整的路径文字描述，
   文字量非常大，表格单元格被撑得很高，难以快速浏览。

   建议：用结构化的卡片或步骤列表替代纯文字表格，
   或者直接链接到计算器预填对应参数的页面。

5. 关于页面（About）缺少版本更新的视觉时间线
   现状：版本历史是纯文字列表，用日期和版本号分隔。

   建议：用可视化的时间线（timeline）组件展示版本历史，更直观。


【八、缺失的功能】

1. 没有计算历史记录展示
   现状：代码中有 history 状态，每次计算都会记录到 state.history 中，
   但这个历史记录从未在 UI 中展示给用户。
   用户无法回顾之前的计算结果。

   建议：添加历史记录面板或页面，展示最近的计算记录，
   支持点击历史记录快速恢复之前的输入和结果。

2. 没有深色模式
   现状：只有浅色主题，蓝白配色。
   部分用户（尤其是夜间使用）可能偏好深色模式。

   建议：添加深色模式切换，使用 CSS 变量实现主题切换。
   可以跟随系统设置（prefers-color-scheme）自动切换。

3. 缺少新手引导
   现状：第一次使用的用户面对 14 个开关和多个输入框，
   可能不知道从哪里开始。虽然有"注意事项"页面，
   但需要用户主动跳转去阅读。

   建议：首次访问时显示简单的引导提示（tooltip tour），
   指引用户完成第一次计算。

4. 没有键盘快捷键
   现状：所有操作都需要鼠标点击。

   建议：支持 Enter 键触发计算，
   左右方向键切换路径等基本快捷键。


================================================================================
总结：用户体验改造优先级排序
================================================================================

第一优先级（对用户感知影响最大）：
  □ 路径结果可视化重构 — 卡片式布局 + 颜色区分获得/花费 + 路径摘要
  □ Sidebar 抽取为共享 Layout + 当前页面高亮
  □ 设置面板语义统一 + 开关文字优化（不截断）
  □ 输入区域添加"交换"按钮和实时差值反馈
  □ 修复 footer 与 sidebar 宽度不匹配

第二优先级（提升易用性）：
  □ 添加"恢复默认设置"按钮
  □ 添加"重置输入"按钮
  □ 限制条件输入框添加"留空=不限制"提示
  □ 设置变更后提示"请重新计算"
  □ 路径切换改进（显示编号 + 列表模式可选）
  □ 进度条改为诚实的等待动画
  □ 计算成功/失败的视觉反馈优化

第三优先级（新增功能）：
  □ 复制结果 / 分享功能
  □ 计算历史记录展示
  □ 设置预设方案（仅升级/仅理智/全部开启）
  □ 深色模式
  □ 键盘快捷键（Enter 计算，方向键切换路径）
  □ 新手引导

第四优先级（子页面优化）：
  □ Note 页面改为折叠式 FAQ
  □ Data 页面数据从 DataService 动态生成
  □ Data 页面默认展开第一个面板
  □ 推荐路径表格改为结构化卡片
  □ About 页面版本历史改为时间线组件

第五优先级（无障碍与细节）：
  □ 开关控件添加 ARIA 标签
  □ 稀有度增加非颜色区分方式
  □ 弹窗焦点管理 + Escape 关闭
  □ "返回顶部"按钮滚动后才显示
  □ 移动端导航改为汉堡菜单
  □ 输入框改为响应式宽度

注意：第一优先级中的"路径结果可视化"是 2.0 版本最能让用户感受到
"焕然一新"的改动，建议作为 UI 重构的核心突破点。
